# iPad Camp Fire

## 概要

CIRCUS TOKYO（3/27）でのクラブイベント演出用Webアプリケーション。
20台のiPadで動作し、超音波信号によって画面表示を同期的に切り替える参加型映像演出システム。

## 技術スタック

- Next.js（またはVite + React）
- WebGL / GLSLシェーダー（エフェクト処理）
- getUserMedia（端末カメラ取得）
- HTML5 Video（VJ映像再生）

## 画面モード（8種）

キーボード1-8で切替（後に超音波信号トリガーに差し替え予定）。
切替時のトランジションはクロスフェード。

| キー | モード              | 内容                                                                                            |
| ---- | ------------------- | ----------------------------------------------------------------------------------------------- |
| 1    | カメラ+サーモ       | 端末カメラ映像にサーモグラフィ（赤基調）シェーダーを適用                                        |
| 2    | カメラ+フラッシュ   | 端末カメラ映像をランダム周期でホワイトアウト⇔映像                                               |
| 3    | VJ映像+サーモ       | VJ映像にサーモグラフィシェーダーを適用                                                          |
| 4    | VJ映像+フラッシュ   | VJ映像をランダム周期でホワイトアウト⇔映像                                                       |
| 5    | 白単色              | 画面全体を白                                                                                    |
| 6    | 赤単色              | 画面全体を赤                                                                                    |
| 7    | 黒単色              | 画面全体を黒                                                                                    |
| 8    | Sin波トランジション | クエリパラメータ `id=N` に基づく固有delayで位相をずらし、白⇔赤をsin関数で滑らかにトランジション |

## 映像ソース

- **端末カメラ**: `navigator.mediaDevices.getUserMedia({ video: { facingMode: 'environment' } })`
- **VJ映像**: 開発時は `public/dummy-video.mov` を使用。本番ではBroadcastBox（WebRTC）経由のライブ映像に差し替え予定

## クエリパラメータ

各iPadに異なるIDを付与：`?id=0`, `?id=1`, ... `?id=19`

- モード8のsin波トランジションで、IDごとに固有のdelay（位相差）を生成
- 将来的に他のモードでもID依存の演出に使う可能性あり

## サーモグラフィシェーダー仕様

- 入力映像の輝度を取得
- 赤を基調としたカラーマップに変換（暗い→黒/暗赤、明るい→赤/オレンジ/白）
- リアルタイム処理（60fps目標）

## ホワイトアウトフラッシュ仕様

- ランダムな間隔（例: 0.5〜3秒）で画面を白に切り替え
- 白の持続時間も短くランダム（例: 0.05〜0.3秒）
- フラッシュのタイミングは各端末で独立（同期不要）

## 信号切替アーキテクチャ

- 現在: キーボードイベント（1-8）でモード切替
- 将来: 超音波信号による切替に差し替え
- 設計方針: 信号受信部分を抽象化し、トリガーソースを差し替え可能にする
- 信号数は今後増える可能性あり。モード追加が容易な設計にする

## トランジション

- モード切替時はクロスフェード（duration: 0.5〜1秒程度）
- 現在のモードと次のモードを同時にレンダリングし、opacity で遷移

## 開発の進め方

1. プロジェクト初期化（Vite + React + TypeScript）
2. カメラ取得・ダミー映像再生の基盤
3. サーモグラフィシェーダー実装
4. ホワイトアウトフラッシュ実装
5. 単色モード・sin波モード実装
6. キーボードトリガーによるモード切替 + クロスフェード
7. 動作確認・調整

## 注意事項

- iPadのSafariで動作すること（WebGL対応確認）
- カメラアクセスはHTTPS必須（開発時はlocalhost可）
- パフォーマンス重視：20台同時稼働でも安定動作
- 本番は3/27。開発期間は3日
